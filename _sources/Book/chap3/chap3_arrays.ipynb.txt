    "Strings, Lists, Arrays, and Dictionaries\n",
    "========================================\n",
    "\n",
    "The most import data structure for scientific computing in Python is the\n",
    "**NumPy array**. NumPy arrays are used to store lists of numerical data\n",
    "and to represent vectors, matrices, and even tensors. NumPy arrays are\n",
    "designed to handle large data sets efficiently and with a minimum of\n",
    "fuss. The NumPy library has a large set of routines for creating,\n",
    "manipulating, and transforming NumPy arrays. NumPy functions, like\n",
    "`sqrt` and `sin`, are designed specifically to work with NumPy arrays.\n",
    "Core Python has an array data structure, but it's not nearly as\n",
    "versatile, efficient, or useful as the NumPy array. We will not be using\n",
    "Python arrays at all. Therefore, whenever we refer to an \"array,\" we\n",
    "mean a \"NumPy array.\"\n",
    "\n",
    "**Lists** are another data structure, similar to NumPy arrays, but\n",
    "unlike NumPy arrays, lists are a part of core Python. Lists have a\n",
    "variety of uses. They are useful, for example, in various bookkeeping\n",
    "tasks that arise in computer programming. Like arrays, they are\n",
    "sometimes used to store data. However, lists do not have the specialized\n",
    "properties and tools that make arrays so powerful for scientific\n",
    "computing. So in general, we prefer arrays to lists for working with\n",
    "scientific data. For other tasks, lists work just fine and can even be\n",
    "preferable to arrays.\n",
    "\n",
    "**Strings** are lists of keyboard characters as well as other characters\n",
    "not on your keyboard. They are not particularly interesting in\n",
    "scientific computing, but they are nevertheless necessary and useful.\n",
    "Texts on programming with Python typically devote a good deal of time\n",
    "and space to learning about strings and how to manipulate them. Our uses\n",
    "of them are rather modest, however, so we take a minimalist's approach\n",
    "and only introduce a few of their features.\n",
    "\n",
    "**Dictionaries** are like lists, but the elements of dictionaries are\n",
    "accessed in a different way than for lists. The elements of lists and\n",
    "arrays are numbered consecutively, and to access an element of a list or\n",
    "an array, you simply refer to the number corresponding to its position\n",
    "in the sequence. The elements of dictionaries are accessed by \"keys\",\n",
    "which can be either strings or (arbitrary) integers (in no particular\n",
    "order). Dictionaries are an important part of core Python. However, we\n",
    "do not make much use of them in this introduction to scientific Python,\n",
    "so our discussion of them is limited."
   ]

    "### Differences between lists and arrays\n",
    "\n",
    "While lists and arrays are superficially similar---they are both\n",
    "multi-element data structures---they behave quite differently in a\n",
    "number of circumstances. First of all, lists are part of the core Python\n",
    "programming language; arrays are a part of the numerical computing\n",
    "package NumPy. Therefore, you have access to NumPy arrays only if you\n",
    "load the NumPy package using the `import` command.\n",
    "\n",
    "Here we list some of the differences between Python lists and NumPy\n",
    "arrays, and why you might prefer to use one or the other depending on\n",
    "the circumstance.\n",
    "\n",
    "-   **The elements of a NumPy array must all be of the same type**,\n",
    "    whereas the elements of a Python list can be of completely different\n",
    "    types.\n",
    "-   **NumPy arrays support \"vectorized\" operations** like\n",
    "    element-by-element addition and multiplication. This is made\n",
    "    possible, in part, by the fact that all elements of the array have\n",
    "    the same type, which allows array operations like element-by-element\n",
    "    addition and multiplication to be carried out by very efficient C\n",
    "    loops. Such \"vectorized\" operations on arrays, which includes\n",
    "    operations by NumPy functions such as `numpy.sin` and `numpy.exp`,\n",
    "    are much faster than operations performed by loops using the core\n",
    "    Python `math` package functions, such as `math.sin` and `math.exp`,\n",
    "    that act only on individual elements and not on whole lists or\n",
    "    arrays.\n",
    "-   **Adding one or more additional elements to a NumPy array creates a\n",
    "    new array and destroys the old one.** Therefore it can be very\n",
    "    inefficient to build up large arrays by appending elements one by\n",
    "    one, especially if the array is very large, because you repeatedly\n",
    "    create and destroy large arrays. By contrast, elements can be added\n",
    "    to a list without creating a whole new list. If you need to build an\n",
    "    array element by element, it is usually better to build it as a\n",
    "    list, and then convert it to an array when the list is complete. At\n",
    "    this point, it may be difficult for you to appreciate how and under\n",
    "    what circumstances you might want build up an array element by\n",
    "    element. Examples are provided later on: for an example see the\n",
    "    section on `loopingarrays`."
   ]
  },
